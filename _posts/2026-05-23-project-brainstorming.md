I always want to be working on some computer science-related project, but I struggle with committing to something. Most weekends when I sit down to code something, it becomes a one-off project that I never want to go back to. At the moment I do have a project that I’m really excited about, but I’m doing it during an Independent Study class at school next year, so I can’t really start it now. Here I’m just going to try to put out some project ideas I have for the future so that I hopefully feel more obligated to work on them. Now that summer is coming around, though, I should have more time to do stuff like this. I also am planning (as I’ve boldly declared at the top of this blog) that I want to write 100 blog posts this summer. I don’t really know if that’s realistic with everything else I’m promising myself I’ll do, but we’ll see. I’m aiming for a mix of computer science and whatever else is on my mind, like running or cooking or music. Anyway, here goes with the brainstorming:

The project I’ll be doing for school is an image processing “library”/algorithm/suite, and I really want to work up to a [pose estimator](https://en.wikipedia.org/wiki/3D_pose_estimation) that I can use for processing running data from video. I’m really psyched about this idea, and I think the formal academic environment of school will let me really make progress on it.

Going along with the theme of running, I think it would be cool to create some sort of “team dashboard” or stats viewer for cross country. We have big ambitions for the state championship this year, and it would be cool to know where we stand. One of my teammates last year made a cool spreadsheet tracking other teams, which I could automate with web scraping and create a cool website for. I’ve looked into this in the past, but Athletic.net and Milesplit both prohibit web scraping, so I’ll have to figure something out.

I think it would also be good for me to go back to my soft body physics engine now that I’ve taken physics and linear algebra; whatever I was doing before concept-wise was a little scrappy. I’m really interested in physics simulation in general, so I could expand it to more types of entities.

Lastly, I want to learn a bit more about backend and network stuff, so I might do a little project just to learn that. But honestly I’m having trouble coming up with more larger-scale ideas, and I might also do some smaller projects for fun relating to cooking or music. Either way, I think this is a decent start and a good way to get some ideas flowing; I’ve already started working a bit more on my physics engine again.
